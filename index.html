快速配置

推荐优先使用 dns.ihccc.com 域名接入；若需固定 IP，请将示例替换为实际节点。

Windows 10/11

  1. 设置 → 网络和 Internet → 更改适配器选项。
  2. 右键当前网络 → 属性 → Internet 协议版本 4/6。
  3. 选择“使用下面的 DNS 服务器地址”，填入 由 dns.ihccc.com 解析得到的实际 IP。若使用 Windows 11 DoH，可在“DNS 设置”中直接填写 HTTPS 端点。

macOS

  1. 系统设置 → 网络 → 选中网络 → 详情。
  2. DNS → 添加服务器，输入 解析自 dns.ihccc.com 的 IP（该字段要求 IP）。
  3. 如需 DoH，可在浏览器或守护进程中设置 https://dns.ihccc.com/dns-query。

iOS / iPadOS

  1. 设置 → Wi-Fi → 点击当前网络的 “i”。
  2. 配置 DNS 选择“手动”，删除旧地址后添加 从 dns.ihccc.com 获取的 IP（此处需 IP）。
  3. 支持 DoH 的 App（如 AdGuard）可直接填入 HTTPS 端点，无需填写 IP。

Android 9+

  1. 系统设置 → 网络和互联网 → 私有 DNS。
  2. 选择“指定主机名”，输入 dns.ihccc.com（DoT）。
  3. 保存后全局生效，无需单独填写 IP。

路由器 / NAS

  1. 在 LAN DHCP 或 WAN DNS 处填写 解析自 dns.ihccc.com 的 IP；仅当设备支持 DNS 主机名时才可直接填域名。
  2. 若设备支持 DoH/DoT，推荐使用加密端点减轻劫持。
  3. 确保允许 53 / 853 / 443 出站端口。

服务承诺

  • 隐私优先：不注入广告或修改解析结果，最小化必要的运行日志。
  • 加密可选：DoH / DoT 端点默认开放，便于规避劫持与探测。
  • 高可用：支持多节点扩展，域名自动指向可用节点，未来可按区域智能分配。
HTTP/2 & HTTP/3 EDNS Client Subnet 可选 零劫持 可自托管部署

FAQ

需要填 IP 还是直接填域名？
优先推荐直接使用 dns.ihccc.com，后端会根据节点状态自动分配。只有在特定场景（如离线环境）才需要固定 IP。
DoH 端点与路径可以改吗？
默认端点为 https://dns.ihccc.com/dns-query。如需自定义路径或多租户，可以在部署时修改并在此页面同步公布。
